# 0.1° 风光场站出力计算方案（Plan）

本文档说明 **0.1° 分辨率** 场站选址结果下的逐时出力计算方案，区别于原有 1° 版本
（见 `station_output-guide.md`）。对应新代码：`station_output_calculator_0p1deg.py`。

---

## 0. 与 1° 版本的核心差异

| 项目 | 1° 版本（旧） | 0.1° 版本（本方案） |
|------|--------------|---------------------|
| 场站含义 | 一个 1°×1° 格元，内含 ~100 个 0.1° CF 格点 | 一个 **0.1° 格元**（单点） |
| 场站经纬度 | `[-179.5, 179.5]`，步长 1°（半度中心） | **0.1° 整数偏移网格中心**，如 24.0, 24.1, -175.2 |
| CF 与场站关系 | CF 更精细，需 **bbox 内空间聚合** | 二者尺度相同，**一一对应（最近邻取点）** |
| 国界内 CF 子格点筛选 | 需要（聚合时只取国界内格点） | **不再需要**（单点取值） |
| 聚合方式 | `mean(CF in bbox ∩ country)` | 取场站中心 **最近的单个 CF 格点** 的时间序列 |

> 关键转变：1° 版本是"降尺度聚合"，0.1° 版本是"同尺度配准取点"。

## 1. 输入数据

### 1.1 场站选址 CSV（0.1°）

路径：`data/stations/stations_SSP*.csv`，列与旧版相同：

```csv
year,type,lon,lat,capacity_gw
2050,solar,24.00,59.00,0.0060
2050,solar,24.10,59.00,0.0055
```

经实测（`stations_SSP1-2.6.csv`，287,652 行）：

- `lon` ∈ [-175.2, 178.3]，`lat` ∈ [-53.2, 68.8]
- `lon`/`lat` 均为 **0.1° 的整数倍**（偏移量 0，即落在 X.0/X.1/X.2… 上），
  实测距最近 0.1° 倍数的最大偏差 < 1e-13 → 严格的规则 0.1° 网格中心。
- `year` ∈ {2030, 2040, 2050}，`type` ∈ {solar, wind}
- 仍是三阶段嵌套优化结果，同一 `(lon, lat, type)` 可能跨年份重复 →
  **去重 + 记录 `activation_year`**（逻辑与旧版完全一致）。
- 去重后 `(lon, lat, type)` 唯一组合约 164,349 个（SSP1-2.6）。

### 1.2 容量因子（CF）网格规格 —— 与场站网格的一致性检查

这是本任务最关键的一步。实测三类 CF 数据源的网格：

| 数据源 | CF 经度示例 | CF 纬度示例 | 步长 | 与 0.1° 场站网格是否对齐 |
|--------|-------------|-------------|------|--------------------------|
| **BCSD** (Germany) | 5.0, 5.1, 5.2 … | 47.0, 47.1 … | 规则 0.1° | **对齐**（整数偏移，同场站网格） |
| **China** | 70.05, 70.15, 70.25 … | 15.05, 15.25, 15.35 …（步长 0.1/0.2 混合） | **不规则 + 偏移 0.05°** | **不对齐** |
| **NAM-12** | 2D 旋转极网格 `lat(rlat,rlon)`/`lon(rlat,rlon)` | 同左 | ~0.11° | **不对齐**（非规则经纬网） |

结论：

- **BCSD**：CF 与场站网格**严格对齐**（都是整数偏移的 0.1° 倍数）。最近邻取点 ≈ 精确匹配，
  理论最近邻距离 ≈ 0。
- **China**：CF 中心点比场站中心点偏移 **0.05°**，且纬向步长不规则（0.1°/0.2°）。
  **不对齐**，最近邻距离约 0.05–0.11°。
- **NAM-12**：旋转极 2D 网格，与规则经纬网本就不对齐，最近邻距离约 0.05–0.08°。

### 1.3 网格不一致时的处理方案（通用做法）

采用 **最近邻取点（nearest-neighbor sampling）**，对三类数据源统一适用：

1. 对每个场站中心 `(lon, lat)`，在 CF 网格中查找**距离最近的单个格点**，
   将该格点的整条时间序列作为该场站的 CF。
2. **规则经纬网（BCSD/China）**：经纬可分离，分别在 1D `lat`、1D `lon` 上做最近邻
   （`lat` 单调用 searchsorted / argmin；`lon` 用环形距离处理 0°/360° 衔接），
   时间复杂度 O(log N)。
3. **旋转极网（NAM-12）**：将 2D `lat`/`lon` 展平，用 `scipy.spatial.cKDTree`
   （无则退化为分块 argmin）查询最近格点。
4. **距离容差 `--max-dist`（默认 0.15°）**：若场站到最近 CF 格点的距离 > 容差，
   认为该场站落在 CF 覆盖范围之外（或网格缝隙），出力记为 NaN 并计数告警。
   - BCSD 对齐：几乎所有距离 ≈ 0，不会触发。
   - China/NAM-12：正常距离 0.05–0.11°，在容差内；只有真正出界的才被剔除。
5. **取点后若该 CF 格点本身为 NaN**（如海洋上的 CF 缺测），出力为 NaN。
6. 不再做"国界内 CF 子格点筛选"——单点取值无此需求。

> 说明：因为场站与 CF 已是同一 0.1° 量级，最近邻是最直接且物理合理的配准方式，
> 不引入额外平滑。若日后需要更稳健，可改为"双线性插值/邻域均值"，本方案先用最近邻。

### 1.4 场站→国家/数据源的归属（仍需 shapefile）

虽然不再做 CF 子格点筛选，但仍需把每个场站分配到国家，原因：

- **BCSD** 的 CF 文件是**按国家**组织的（`{model}/{region}/...nc`），
  必须先判定场站属于哪个国家，才能选对 CF 文件。
- 判定方式与旧版一致：场站中心点对 Natural Earth 国界做 **point-in-polygon**。
- China 数据源：所有落在中国境内的场站；NAM-12：北美三国。

国家名映射（`South-Africa`→`South Africa` 等）沿用旧版 `BCSD_REGION_TO_NAME`。

---

## 2. 年份匹配与激活年份

与旧版完全一致：

1. 按 `(lon, lat, type)` 去重，最早年份作为 `activation_year`。
2. 出力仅在 `CF年份 >= activation_year` 时有效，否则 NaN。

---

## 3. 出力计算

```
CF_station(t) = CF[nearest_grid_point](t)          # 最近邻取点（非聚合）
power(t)      = CF_station(t) × capacity_gw         # 单位 GW
```

- `CF年份 < activation_year` → NaN
- 最近 CF 格点距离 > 容差，或该点 CF 为 NaN → NaN

---

## 4. 输出格式

与旧版 **完全一致**（便于下游复用），仅文件可加 `_0p1deg` 区分（见"开放问题"）：

- 维度：`time`、`id`
- 变量：`power(time,id)`、`time`、`station_lon`、`station_lat`、
  `station_type`(0=光伏/1=风电)、`capacity_gw`、`activation_year`
- 全局属性：`region`、`scenario`、`source_csv`；NAM-12 额外 `gcm`/`realization`/`rcm`
- 额外建议属性：`grid_resolution="0.1deg"`、`match_method="nearest"`、`max_match_dist_deg`

输出路径默认沿用 `outputs/pv_out/...`、`outputs/wind_out/...`、`outputs/*_NAM-12/...`，
为避免与 1° 结果混淆，**默认输出根目录为 `outputs/outputs_0p1deg`**（1° 版本为
`outputs/outputs_1deg`；可用 `--output-dir` 覆盖）。

## 5. 实现要点

1. 复用旧代码骨架（CLI、CF 文件查找 `find_cf_file*`、国家边界加载、断点续跑 `--overwrite`、
   保存逻辑）。
2. 改写计算核心：把"bbox 聚合"替换为"最近邻取点"。
   - 预处理：为每个场站求 `(lat_idx, lon_idx)`（规则网）或展平索引（NAM-12）+ 距离 + 有效标志。
   - 分块按时间读取 CF 整张空间切片，向量化 `cf_chunk[:, lat_idx_arr, lon_idx_arr]`
     一次性取出所有场站的 (chunk, n_stations) 序列。
3. 经度统一：场站 `lon % 360` 对齐 CF 的 0–360；国界 contains 用 `lon→[-180,180]`。
4. 性能：China 全网 348×442、chunk=1000 → 单块 ~0.6GB，可接受；BCSD 按国小网更快。
5. 新增参数：`--max-dist`（默认 0.15）。

## 6. 开放问题 / 待用户确认（先按通用默认执行，不阻塞）

1. **匹配方法**：默认采用"最近邻单点取值"。是否需要改为邻域均值/双线性插值？
   （默认：最近邻——因场站与 CF 同为 0.1° 量级，最直接。）
2. **距离容差**：默认 `0.15°`。China/NAM-12 偏移最大约 0.11°，0.15° 可覆盖；
   是否需要调整或对不同数据源设不同容差？（默认：统一 0.15°。）
3. **输出目录**：默认写到 `outputs/outputs_0p1deg/`，避免覆盖 1° 结果。是否要同目录加文件后缀？
   （默认：独立目录 `outputs/outputs_0p1deg`。）
4. **新出现的 `CFs_of_*_ERA5Land/`**（按月份的观测 CF，无 SSP 情景）是否纳入？
   （默认：**不纳入**——无情景维度，且任务针对 SSP 场站。）
5. **`data/grid_division/`（20 地理区域 tif + json）**：用户明确表示**暂不处理**，本方案忽略。
6. **BCSD 国家边界外、但在 CF bbox 内的场站**：最近邻仍可能取到邻国/海洋格点。
   默认行为：只要距离在容差内且 CF 非 NaN 就取值（不再二次用国界裁剪 CF 点）。
   场站本身已通过 point-in-polygon 归属到该国，故影响极小。
7. **China 不规则纬度**：`lat` 步长 0.1/0.2 混合但单调递增，最近邻用 1D argmin 稳健，
   不假设固定步长。
8. **跨经度选择数据源冲突**：同一场站理论上可能既在 BCSD 某国又在 China/NAM-12 范围内。
   沿用旧版：由 `--source` 指定一次只跑一类数据源，互不干扰。
